Om te testen of Jules de **Bootstrap Data** correct heeft begrepen zonder de scripts opnieuw te scannen, kun je de volgende prompt gebruiken. Deze prompt dwingt hem om gebruik te maken van de "Registry" die door je script is aangemaakt.

### De Test Prompt
````markdown
Based **only** on the `ASP SKILL REGISTRY` provided during startup, generate a Markdown table of all available skills. Use the following format: 

```markdown
| skill | description | triggers |
| --- | --- | --- |
```

Do not perform a new directory scan or use `list_files`. If a description or trigger is missing from the registry, mark it as 'Not defined in metadata'."
````


---

### Wat deze prompt controleert:

1. **Strict Mode:** Door te zeggen "Based **only** on the registry", test je of hij de data uit de `bootstrap.py` heeft onthouden.
2. **No Scanning:** Als je in de logs van Jules ziet dat hij géén `list_files` of `read_file` uitvoert op de `/._/skills/` mappen, dan werkt je `AGENTS.md` instructie perfect.
3. **Data Integrity:** Het laat zien of je `parse_yaml_frontmatter` functie in de Python-bootstrap de juiste velden (zoals 'trigger') correct heeft geëxtraheerd.

### Wat als Jules zegt dat hij niets weet?

Als Jules antwoordt dat hij geen skills kent, dan is de output van je `bootstrap.sh` niet correct in zijn **System Context** terechtgekomen. In dat geval moeten we controleren of de omgeving de `stdout` van je startup-script wel doorgeeft aan de chat-sessie.
